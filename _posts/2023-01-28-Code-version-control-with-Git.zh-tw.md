# 簡介

程式碼版本控制是一個允許軟體開發人員追蹤原始碼變更的過程。它使開發人員在出現問題或需要進行更改時，能夠輕鬆地回滾到以前的版本。版本控制還允許多位開發人員在同一個程式碼庫上工作，而不會互相覆蓋彼此的工作。

Git 是當今開發人員最常用的版本控制系統之一。它快速、可靠且功能豐富。在本文中，我們將探討如何使用 Git 進行程式碼版本控制。

# 設定 Git

在使用 Git 之前，您需要下載並安裝它。您可以在 https://git-scm.com/downloads 找到下載頁面。下載安裝程式後，只需按照說明完成安裝即可。

# 建立儲存庫

安裝 Git 後，您需要建立一個儲存庫（Repository）。儲存庫是由 Git 追蹤的文件和目錄的集合。要建立儲存庫，您需要開啟終端機視窗並執行以下命令：

```
git init
```

這將在當前目錄中建立一個新的儲存庫。

# 將檔案新增至儲存庫

現在您有了一個儲存庫，您需要將檔案新增到其中。要將檔案新增到儲存庫，您需要執行以下命令：

```
git add <檔案名稱>
```

這將把指定的檔案新增到儲存庫中。您可以使用以下命令一次新增多個檔案：

```
git add *.<副檔名>
```

這將把所有具有指定副檔名的檔案新增到儲存庫中。

# 提交變更

將檔案新增到儲存庫後，您需要提交（Commit）您的變更。要提交變更，您需要執行以下命令：

```
git commit -m “<訊息>”
```

這將提交您所有的變更，並附上指定的訊息。

# 推送到遠端

提交變更後，您需要將它們推送到遠端儲存庫。為此，您需要執行以下命令：

```
git push <遠端名稱> <分支名稱>
```

這將把您的變更推送到指定的遠端儲存庫和分支。

# 從遠端拉取

如果您需要從遠端儲存庫拉取（Pull）變更，可以使用以下命令：

```
git pull <遠端名稱> <分支名稱>
```

這將從指定的遠端儲存庫和分支拉取任何變更。

# 總結

Git 是一個強大的版本控制系統，允許開發人員追蹤和管理原始碼的變更。它快速、可靠且功能豐富。在本文中，我們介紹了如何使用 Git 進行程式碼版本控制。
