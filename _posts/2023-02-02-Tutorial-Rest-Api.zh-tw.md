# 簡介

REST (具象狀態傳輸) API 是一種基於 Web 的架構，能夠讓不同系統透過網際網路進行通訊。它是一組允許程式相互對話的規則。REST API 定義了一組開發人員可以用來執行請求和接收回應的函數。

# 什麼是 REST API？

REST API 是一種應用程式介面 (API)，它使用 HTTP 請求來執行數據的 GET、PUT、POST 和 DELETE 操作。它是一種基於 Web 的架構，能夠讓不同系統透過網際網路進行通訊。它是一組允許程式相互對話的規則。REST API 定義了一組開發人員可以用來執行請求和接收回應的函數。

# 如何使用 REST API

使用 REST API 非常簡單。您只需要向 API 端點 (endpoint) 發送帶有適當參數的 HTTP 請求即可。API 隨後會以 HTTP 回應碼和數據負載 (payload) 的形式返回回應。

REST API 中最常用的 HTTP 方法是 GET、POST、PUT 和 DELETE。這些方法中的每一種都有特定的用途，並用於執行不同類型的操作。

# GET

GET 方法用於從伺服器檢索數據。它是最常用的方法，用於從特定資源獲取數據。

# POST

POST 方法用於在伺服器上建立新資源。它用於將數據發送到伺服器並建立新資源。

# PUT

PUT 方法用於更新伺服器上的現有資源。它用於將數據發送到伺服器並更新現有資源。

# DELETE

DELETE 方法用於從伺服器刪除資源。它用於向伺服器發送請求以刪除特定資源。

# 結論

REST API 是開發人員建立可與網際網路上其他系統通訊的 Web 應用程式的強大工具。它是一組允許程式相互對話的規則，用於執行請求和接收回應。REST API 中最常用的 HTTP 方法是 GET、POST、PUT 和 DELETE。這些方法中的每一種都有特定的用途，並用於執行不同類型的操作。透過 REST API，開發人員可以輕鬆建立與其他系統通訊的 Web 應用程式。
